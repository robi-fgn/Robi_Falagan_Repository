◇ Node.js

Node.js is a JavaScript runtime environment that allows developers to run JavaScript code outside of the browser.It provides access to powerful package managers, which make it easy to install and manage libraries.
Tools that require Node.js are part of the modern web development ecosystem, offering flexibility and automation — but they might require setup, command-line use, and sometimes higher system resources, which could seem like a barrier for beginners.

TailwindCSS

Tailwind CSS is a utility-first framework in which one can apply predefined styles directly into the HTML. Its compiler (JIT) generates only the classes used, therefore reducing the final CSS size. It allows for the building of consistent design systems with a professional look, avoiding the task of writing custom CSS from scratch.

Learning curve: Medium (requires learning each class name)
Use: Modern websites and apps

Astro

Astro is a content-driven framework that works with island architecture. This implies the priorization of static content, while having the flexibility to introduce specific interactive elements when needed. Only the interactive elements load JavaScript, improving its performance.

Learning curve: Medium
Use: Static marketing sites, websites with selective interactivity.

◇ No runtime programme

Tools in this category do not require any special runtime environment to function. They can run directly in the browser or by simply linking their CSS or JavaScript files from a CDN, becoming lightweight, accessible, and ideal for quick prototyping or student projects like this one. These frameworks are easy to learn and implement, and require no installation or messy learning curve.

Bootstrap

Bootstrap is a classic framework that works with a grid system and ready-to-use components. It doen't need Node.js or any other runtime programme, it only requires the linking of its CSS and JS files.

Learning curve: Beginner friendly
Use: Fast prototypes and educational work

GSAP

GSAP (GreenSock Animation Platform) is a highly optimized JavaScript library. It allows for a precise control of animations through the use of timelines, scroll triggers and sequence animations. It has the downside of being limited to simple animations as it isn't a full framework in itself.

Learning curve: Medium - High (there are various controls that might be complex)
Use: Portfolios and web presentations
No Runtime

Go to website

AOS

AOS (animate on Scroll), as the name suggests, is an animation library that offers a variety of scroll-triggered animation codes, ready for the user to copy and paste into their project. AOS doesn't need Node.js, it works through a connection to a CDN (Content Delivery Network).

Learning curve: Easy - Medium
Use: Static websites that may need dynamism

◇ Alternative runtime programmes

Some tools don't rely on Node.js but use other runtime environments or architectures. Self-contained executables, cloud-based services and different programmes like Ruby are all alternatives that prove web development doesn't depend solely on Node.js.

Jekyll

Jekyll is a classic static site generator that converts plain text from Markdown files into static websites through the use of liquid templates. It's simple and easy to integrate to Github, but it lacks interactivity.

Learning curve: Easy - Medium
Use: Personal or simple educational projects that don't need interactivity

Hugo

Hugo is an ultra-fast static site generator that, just like Jekyll does, uses Markdown files to create static websites. It has a simple instalation process and its standalone binary runtime doesn't require the use of Node.js nor any other programme.

Learning curve: Easy - Medium
Use: Blogs and personal projects

Hygraph is a headless CMS (Content Management System) that works in the Cloud and allows for the creation of content through API GraphQL, without installing any local software. It's a framework that allows dynamicity in a static site.

◇ Glossary

CDN: A content delivery network is a group of geographically distributed servers that speed up the delivery of web content by bringing it closer to users.

CMS: A content management system is software that helps users create, manage, store, and modify their digital content

headless CMS: A modern content management system that separates content creation (backend) from content presentation (frontend) using APIs. This separation allows your content to be delivered across any platform — from websites to mobile apps, smart devices, and beyond.

Island architecture: a pioneering approach to web development that aims to enhance performance by combining concepts from server-side rendering, static site generation, and partial hydration. It encourages small, focused chunks of interactivity within server-rendered web pages.

SaaS: Software as a Source is a method of software delivery in which software is accessed online (in the Cloud) via a subscription, rather than bought and installed on individual computers.

Standalone binary: A self-contained executable program that can be run directly on computer without the need of any runtime.
